## 要构建什么

完善子代理的各种边界情况和健壮性处理。

1. **输出截断**：子代理最终输出在返回父代理前，按 50KB / 2000 行阈值执行头部截断（`truncateHead`）。截断后在输出末尾追加截断提示信息（显示行数/字节数等），并写入完整内容到临时文件告知 LLM 可通过 `read` 工具查看。

2. **错误处理强化**：
   - 子代理进程非零退出码：工具结果标记为 `isError: true`，错误信息写入 `output`
   - 子代理进程崩溃（spawn 失败等）：捕获异常，返回友好的错误消息
   - 未知代理名称：抛出错误，并列出所有可用代理名称
   - JSONL 解析失败：非 JSON 行静默跳过（子进程可能输出非 JSON 内容）

3. **大任务处理**：任务描述超过 8000 字符时，写入临时文件并通过 `@task.md` 文件引用语法传递给子进程。需要用到 `withFileMutationQueue` 确保文件写入安全。

4. **临时目录生命周期**：临时目录在 `runSubagent` 结束时通过 `try-finally` 清理，确保即使子进程崩溃也不会留下泄漏的临时目录。

## 验收条件

- [ ] 子代理输出超过 50KB 时被头部截断，截断提示信息出现在输出末尾
- [ ] 截断后的完整内容写入临时文件，输出中包含该文件路径
- [ ] 子代理非零退出时工具结果包含 `isError: true`
- [ ] 传入未知代理名称时抛出包含可用代理列表的错误
- [ ] 子进程输出中的非 JSON 行被静默跳过
- [ ] 任务 >8000 字符时正确写入临时文件并通过 `@` 语法引用
- [ ] 无论子代理成功还是失败，临时目录均被清理
