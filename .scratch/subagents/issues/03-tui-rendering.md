## 要构建什么

实现 subagent 工具的 TUI 自定义渲染，两种视图通过 `ctrl+o` 切换。

**折叠视图（默认）**：

工具调用头部：单行显示 `subagent <agent-name> <60字符任务预览>`。

结果区域从顶向下依次显示：
1. 代理状态行：`✓/<agent> (model) — <工具数> tools · <耗时>`
2. 按时间顺序排列的工具调用日志，每行一个工具调用。运行中的标 `▸`，已完成的标 `  `
3. 最新消息摘要行（取助手消息中非代码块的前 3 段文字）
4. 用量行：`↑<输入> ↓<输出> R<缓存读> W<缓存写> $<费用> <上下文窗口%>/<最大值>`

上下文窗口百分比超过 90% 时标红（`error`），超过 70% 时标黄（`warning`），其余灰色（`dim`）。

**展开视图**：

工具调用头部追加完整流式任务正文（在参数增量传送时同步渲染）。

结果区域除了折叠视图的内容外，额外渲染：
1. 代理的完整最终输出（Markdown 格式渲染）
2. 嵌套子代理（当 worker 派发 scout/researcher 时），内联渲染在发起它们的工具调用行下方，缩进 2 空格，显示各自的状态行、工具调用日志和用量行。

## 验收条件

- [ ] 折叠视图正确显示代理状态行、工具调用日志和用量行
- [ ] 运行中的工具调用显示 `▸` 前缀，已完成的显示 `  ` 前缀
- [ ] `ctrl+o` 切换展开视图后显示完整输出（Markdown 格式）
- [ ] 展开视图中流式任务正文随参数传输实时更新
- [ ] 上下文窗口百分比超过阈值时正确着色
- [ ] 嵌套子代理的进度内联渲染在正确的工具调用行下方
- [ ] 嵌套子代理以 2 空格缩进显示
